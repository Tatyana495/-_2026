# ChemAI Predict the Cure

Решение для хакатона **ChemAI Predict the Cure** по предсказанию биологической активности химических соединений на основе числовых молекулярных дескрипторов.

## Задача

В соревновании необходимо построить ML-пайплайн, который по признаковому описанию химического соединения предсказывает три целевые переменные:

- `IC50, mM` — концентрация вещества, при которой подавляется 50% активности вируса;
- `CC50, mM` — концентрация вещества, при которой наблюдается токсичность для 50% клеток;
- `SI` — индекс селективности соединения.

`SI` связан с `IC50` и `CC50`, однако по условию соревнования он предсказывается как отдельная целевая переменная.

## Данные

В решении используются следующие файлы:

- `train.csv` — обучающая выборка с признаками и целевыми переменными;
- `test.csv` — тестовая выборка без целевых переменных;
- `sample_submission.csv` — шаблон файла для отправки решения.

Внешние данные не используются. `sample_submission.csv` используется только как шаблон итогового файла.

## Структура решения

В ноутбуке реализован полный воспроизводимый пайплайн:

1. загрузка данных;
2. первичный анализ данных;
3. проверка пропусков, дубликатов, константных и почти константных признаков;
4. визуальный анализ распределений целевых переменных;
5. подготовка признаков;
6. обучение отдельных моделей для `IC50`, `CC50` и `SI`;
7. построение OOF-предсказаний по 5-fold cross-validation;
8. взвешенный blend базовых моделей;
9. Ridge stacking;
10. финальная постобработка предсказаний;
11. сохранение итогового файла `submission.csv`.

## Разведочный анализ данных

В ходе анализа было выявлено, что целевые переменные имеют выраженные правые хвосты и сильную асимметрию. Особенно это заметно для `SI`.

Поэтому в решении используется преобразование:

```python
log1p(target)
```

Оно уменьшает влияние экстремально больших значений и делает обучение моделей более устойчивым.

Также были проверены:

- пропуски в признаках;
- полные дубликаты строк;
- дубликаты по молекулярным признакам;
- константные признаки;
- почти константные признаки;
- корреляции между целевыми переменными;
- признаки с наибольшей корреляцией с target.

Полностью константные признаки удаляются, так как они не помогают различать объекты. Почти константные признаки оставлены, поскольку редкие функциональные группы могут содержать полезный химический сигнал.

## Модели

Для каждой целевой переменной обучается отдельный набор моделей. Такой подход выбран потому, что `IC50`, `CC50` и `SI` имеют разные распределения, масштабы и характер ошибок.

Используются следующие модели:

- `ExtraTreesRegressor`;
- `RandomForestRegressor`;
- `HistGradientBoostingRegressor`;
- `Ridge`;
- `CatBoostRegressor`;
- `LightGBM`;
- `XGBoost`.

Модели `CatBoost`, `LightGBM` и `XGBoost` являются дополнительными. Если соответствующие библиотеки не установлены, связанные модели будут пропущены.

Для получения полного результата рекомендуется установить все зависимости.

## Валидация

Качество моделей оценивается с помощью OOF-предсказаний по 5-fold cross-validation.

Для каждой модели рассчитывается OOF RMSE. Затем модели объединяются во взвешенный ансамбль: чем ниже OOF RMSE модели, тем больший вес она получает в итоговом blend.

Дополнительно используется Ridge stacking по OOF-предсказаниям базовых моделей. Stacking позволяет лучше объединить разные алгоритмы и улучшает локальную OOF-метрику по сравнению с простым взвешенным ансамблем.

## Постобработка

После получения предсказаний применяется финальная постобработка:

- обратное преобразование из `log1p`;
- калибровочные множители для `IC50`, `CC50` и `SI`;
- запрет отрицательных значений;
- ограничение верхнего хвоста для `SI`;
- сохранение результата в формате `sample_submission.csv`.

Связь `SI` с отношением `CC50 / IC50` не используется как жёсткая формула. `SI` предсказывается отдельной моделью, что соответствует условию соревнования.

## Финальная конфигурация

Финальная зафиксированная конфигурация:

## Как запустить

1. Поместить файлы данных рядом с ноутбуком:

```text
train.csv
test.csv
sample_submission.csv
```

или указать путь к папке с данными в переменной `DATA_DIR`.

2. Установить зависимости:

```bash
pip install pandas numpy scikit-learn matplotlib catboost lightgbm xgboost
```

3. Открыть ноутбук и выполнить:

```text
Kernel → Restart & Run All
```

4. После выполнения будет создан итоговый файл:

```text
submission.csv
```

## Зависимости

Основные библиотеки:

```text
python
pandas
numpy
scikit-learn
matplotlib
catboost
lightgbm
xgboost
```

Если `catboost`, `lightgbm` или `xgboost` не установлены, ноутбук может выполниться, но итоговые предсказания могут отличаться, так как часть моделей будет пропущена.

## Выходной файл

Финальный ноутбук сохраняет ровно один файл:

```text
submission.csv
```

Перед сохранением промежуточные файлы вида `submission_*` и `candidate_*` удаляются, чтобы снизить риск отправки неправильного файла.

## Воспроизводимость

Для воспроизводимости используются фиксированные значения `random_state`.

Пайплайн не использует:

- внешние данные;
- целевые значения тестовой выборки;
- ручное изменение строк в `submission.csv`;
- признаки, построенные на основе target из test.

Все этапы — загрузка данных, подготовка признаков, обучение моделей, ансамблирование, постобработка и сохранение результата — выполняются внутри ноутбука.
